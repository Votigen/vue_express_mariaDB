### 路由模块化设计解析

1. **userRoutes.js 文件**
   这里我们定义并导出特定于用户的路由：

   ```js
   // src/routes/api/userRoutes.js
   import { Router } from 'express';
   const router = Router();  // 创建路由实例

   // 定义具体路由
   router.get('/', getUsers);
   router.post('/', createUser);

   export default router;  // 导出路由实例
   ```

2. **路由聚合文件 (index.js)**
   这是关键部分 - 我们在 `routes` 目录下创建 `index.js` 作为路由聚合器：

   ```js
   // src/routes/index.js
   import { Router } from 'express';
   import userRoutes from './api/userRoutes';
   import productRoutes from './api/productRoutes';

   const router = Router();

   // 聚合所有子路由
   router.use('/users', userRoutes);
   router.use('/products', productRoutes);

   export default router;  // 导出聚合后的路由
   ```

3. **在 app.js 中使用**
   最后在应用入口文件中引入聚合后的路由：

   ```js
   // src/app.js
   import routes from './routes';  // 自动导入 routes/index.js

   app.use('/api', routes);  // 挂载到 /api 路径
   ```

### 为什么这样设计？

1. **模块化组织**

   - 每个功能模块有自己的路由文件（用户、产品等）
   - 路由定义与控制器逻辑分离
   - 新增功能只需添加新路由文件，无需修改核心文件

2. **路由层级管理**

   - `/api/users` → 用户相关路由
   - `/api/products` → 产品相关路由
   - 清晰的URL结构：`GET /api/users/profile`

3. **单一入口原则**

   - `app.js` 只需导入一个路由聚合器
   - 避免在核心文件中导入大量路由文件

4. **可扩展性**
   添加版本控制非常方便：

   ```js
   // src/routes/index.js
   import v1Routes from './v1';
   import v2Routes from './v2';

   router.use('/v1', v1Routes);
   router.use('/v2', v2Routes);
   ```

### 路由处理流程

```text
客户端请求 → 
app.js (入口) → 
路由聚合器 (src/routes/index.js) → 
子路由 (src/routes/api/userRoutes.js) → 
控制器 (src/controllers/userController.js) → 
服务层 (src/services/userService.js)
```

### 实际请求示例

1. 请求: `GET /api/users`
   - 匹配路径: `/api/users` → 路由聚合器
   - 转发到: `userRoutes` 中的 `router.get('/')`
   - 调用: `getUsers` 控制器
2. 请求: `POST /api/products`
   - 匹配路径: `/api/products` → 路由聚合器
   - 转发到: `productRoutes` 中的 `router.post('/')`

### 为什么优于 express-generator 的方式

传统 express-generator 的路由处理：

```js
// 不推荐的方式 (express-generator 默认)
var usersRouter = require('./routes/users');
app.use('/users', usersRouter);
```

我们的改进：

1. **深层嵌套支持**：可以轻松创建 `/api/v1/admin/users` 这样的路径
2. **统一管理**：所有路由配置在 routes/index.js 中可见
3. **按功能分组**：相关路由放在同一目录
4. **减少核心文件变更**：添加新路由只需修改聚合文件

这种模式是大型 Express 项目的标准实践，它保持了代码的整洁性和可维护性，特别适合团队协作开发。
